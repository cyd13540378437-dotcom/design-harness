# 12 — Mobile Surface Reference Selection

## 前提

- 目标项目存在 `docs/design/reference-library/`。
- 用户请求：“设计一个移动端记录页，用户每天快速记录一次状态。”

## 当

- `design-engineering` 进入视觉方向前的参考检索阶段。

## 则

- 必须先记录 `primary_surface: mobile-app`；若用户明确要求移动 Web，则记录为 `responsive-web` 并说明原因。
- 优先选择移动端记录、习惯反馈、低刺激或快速输入相关参考。
- Web 表格、侧边栏和多列 dashboard 只能作为对象关系参考，不得直接落地布局。
- 用户看到的方向必须是“快速记录 / 陪伴反馈 / 数据回顾”等业务语义，而不是产品名选择。
