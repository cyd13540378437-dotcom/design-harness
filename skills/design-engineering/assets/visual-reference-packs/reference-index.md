# Visual Reference Pack Index

## clean-minimal-product

- 适合：轻量 SaaS、生产力工具、设置页、简单协作工具。
- 视觉特征：大量清晰留白、细边框、低饱和强调色、简洁表单和稳定导航。
- 不适合：需要强情绪表达、内容密度很高或游戏化的产品。
- 配色方向：冷静中性色、单一主强调色、少量语义色。
- 同质化风险：容易变成普通白底 SaaS；需要用产品内容和组件节奏拉开差异。

## dense-operator-console

- 适合：运维后台、数据监控、运营控制台、交易或排班工具。
- 视觉特征：高信息密度、表格、状态色、紧凑工具栏、多列仪表盘。
- 不适合：面向消费者的轻松体验或需要强情感品牌的页面。
- 配色方向：深浅中性基底、明确状态色、控制强调色数量。
- 同质化风险：容易过暗或过度复杂；必须保持层级和可读性。

## warm-creator-tool

- 适合：创作者工具、写作、灵感整理、知识工作台。
- 视觉特征：温暖纸感背景、清晰写作区域、柔和强调、内容优先。
- 不适合：严格金融、工业监控或需要强冷静专业感的系统。
- 配色方向：暖中性色、深色文字、少量泥土色或绿色强调。
- 同质化风险：容易变成浅米色模板；需要控制卡片和装饰。

## editorial-content-system

- 适合：内容产品、媒体后台、文档系统、课程和知识库。
- 视觉特征：编辑式网格、强排版层级、栏目节奏、阅读和管理并重。
- 不适合：纯操作台或需要游戏化即时反馈的应用。
- 配色方向：高对比文字、克制背景、编辑色块用于栏目。
- 同质化风险：容易像营销杂志页；应用内要保持工具效率。

## playful-consumer-app

- 适合：移动消费应用、轻社交、习惯养成、个人体验。
- 视觉特征：明快色块、亲切插画式空状态、触控友好组件、轻量动效感。
- 不适合：严肃企业工具或高密度数据系统。
- 配色方向：明亮主色、柔和背景、有限多彩强调。
- 同质化风险：容易幼稚或过度圆角；要保持产品任务清晰。
