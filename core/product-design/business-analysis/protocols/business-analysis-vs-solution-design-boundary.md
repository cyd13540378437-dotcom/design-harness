# 商业分析与解法设计边界 v0.1.4

## 1. 商业分析负责

```text
当前真正要选择什么；
现实路径和当前推荐；
价值、成本、风险和适用条件；
证据强度与边界；
验证路径的目的、层级和升级条件；
什么会改变或重开决定。
```

## 2. 商业分析不负责

```text
正式数据结构；
具体字段和表格；
评分公式与权重；
样例数据；
脚本或代码；
UI、系统组件和技术栈；
详细操作 Runbook；
实现任务拆分与代执行。
```

## 3. 概念级路线不等于实施设计

可以说明 M0 / M1 / M2 分别降低什么不确定性、何时升级、能证明什么；不能继续定义 M0 的四张表、逐单步骤、评分公式和样例。

## 4. 请求跨界时

保留原 Decision Scope，标记 `out_of_scope_candidate`，只做边界说明，不自动触发其他模块。
