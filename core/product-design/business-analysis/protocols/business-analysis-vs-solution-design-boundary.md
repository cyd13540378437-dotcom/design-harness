# 商业分析与解法设计边界

## 1. 商业分析回答

```text
是否值得做
为什么做
有哪些替代路径
每条路径的价值、成本、风险和证据边界
在什么条件下成立
如何验证
```

## 2. 解法或产品设计回答

```text
具体字段
数据结构
界面
系统组件
算法接口
技术栈
详细功能范围
实现任务拆分
```

## 3. 运行规则

当用户提出“我想做 X”时：

1. 先把 X 记录为候选解法；
2. 比较不做、更小验证、X、本阶段更重解法；
3. 确认是否选择 X；
4. 只有用户继续要求实现细节时，才提供“实施考虑附录”；
5. 附录必须标注为非商业结论、非权威实施规格。

## 4. ARCOS 型示例

商业结论：

> 当前建议选择轻量合成场景策略实验台，用于验证相对策略稳健性。

实施附录：

> 可以用 JSON 驱动、包含工单与资源结构、跑固定随机种子。

后者不能取代前者，也不能在未经确认时成为主结论。
