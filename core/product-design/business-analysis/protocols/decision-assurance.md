# Decision Assurance Check

Decision Assurance Check 是商业分析的输出质量保障层。

它不保证结论一定正确，但保证结论是可审查、可验证、可复审的。

## 1. 结果枚举

```text
pass  可以输出阶段性结论
warn  可以输出临时建议，但必须说明限制
block 不得输出最终建议，只能输出下一步选择
```

## 2. 检查项

阶段性结论前必须检查：

```text
问题是否定义清楚
原始输入是否完整保留
事实、假设、解释是否分开
是否存在真实选项
是否包含不做 / 维持现状 / 先验证
是否进行了反方分析
使用模型是否适配
证据强度是否说明
结论边界是否清楚
是否定义验证方式
是否定义复审条件
是否需要用户明确确认
```

## 3. block 场景

必须 block：

```text
用户输入仍高度歧义
系统未保留原始输入
事实和假设混淆
没有真实选项
缺少反方分析
模型被明显误用
关键输入不足却试图给最终财务结论
高影响决定未经用户确认
```

## 4. warn 场景

可 warn：

```text
证据较弱但可以给 provisional 建议
结论适用范围较窄
当前只能形成验证计划
部分模型输入不完整但没有伪造结论
```

## 5. pass 场景

可 pass：

```text
问题清楚
选项真实
证据和假设分开
模型适配
反方分析完成
验证和复审条件存在
高影响事项已交给用户确认
```


## 6. 独立性检查

质量检查还必须确认：

```text
没有把其他能力的输出当作未经说明的事实；
没有生成跨模块触发或下游任务；
没有修改本模块根目录之外的项目知识资产。
```
