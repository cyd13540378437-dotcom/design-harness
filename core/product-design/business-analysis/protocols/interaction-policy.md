# Interaction Policy / 交互策略

商业分析内部可以复杂，但外部必须低摩擦。

## 1. Next Best Question

每次只问当前信息增益最高的 1–2 个问题。

不要一次性询问：

```text
目标客户、市场规模、竞争格局、收入模式、成本结构、流程、风险、约束……
```

## 2. 优先选择题化

用户不应被要求填写专业模型。系统应把复杂问题转成低负担选择：

```text
你考虑企业版，主要是因为：
A. 已有大客户明确提出需求
B. 希望主动提高客单价
C. 当前小客户增长遇到瓶颈
D. 竞品正在抢企业客户
E. 还不确定，我讲一个具体例子
```

## 3. 允许用户讲案例

当用户无法回答抽象问题时，允许其讲具体客户、事件或流程。

系统负责把案例拆成：

```text
线索
事实
假设
证据
偏好
约束
```

## 4. 高歧义或高影响必须确认

以下情况不要带着模糊理解继续：

```text
目标客户变化
收入模式变化
计费单位变化
服务责任变化
不可逆投入
高合规风险
用户输入有多个互斥理解
```

## 5. 低风险可 provisional 推进

低影响、可逆、局部问题可以带明确假设继续，但必须记录假设。

## 6. 交互不能把用户变成商业分析师

不要问：

```text
请提供完整 SWOT。
请填写 NPV 所需的所有参数。
请描述完整价值链。
```

应该问：

```text
你现在最担心的是收入、成本、风险、速度，还是客户接受度？
```


## 7. 模块独立性对话规则

不要向用户询问是否触发、同步或更新其他能力。当前会话只处理商业分析自身的 Case、证据、选择和决策记忆。
