# Post-decision Clarification / 决定后澄清合同 v0.1.4

## 1. 允许的澄清

选择确认后，商业分析可以解释：

```text
术语或阶段的含义；
推荐理由；
收益、代价和风险；
证据边界；
进入、升级、停止和复审条件；
该方案如何服务原商业选择。
```

## 2. 不允许的扩张

不得继续生产：

```text
逐步操作流程；
具体表格与字段；
公式、评分和权重；
样例数据；
脚本或代码；
系统组件、UI 或技术架构；
实施任务拆分；
执行结果。
```

## 3. 标准响应结构

```text
先确认：本次业务选择已经完成；
再解释：用户询问概念在原方案中的目的和边界；
明确：哪些具体细节已经属于后续实施；
收口：Case 进入 completion-review，不再生成新选择。
```

## 4. ARCOS 的 M0

允许说明：

> M0 是一个不先开发系统的最小方法检查，用于确认两种策略能否在同一条件下比较、指标是否能反映人力利用效率、以及是否值得进入 M1。

不允许继续给出四张 Excel 表、逐单派发步骤、具体评分公式、权重或样例生成。
