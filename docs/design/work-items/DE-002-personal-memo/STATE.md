# Work Item 状态

## 任务摘要

设计并实现一个个人备忘录原型。目标是让用户快速记录、清楚浏览和安全删除备忘，同时保持暖白、干净、低干扰的视觉体验。

## 范围

### 包含

- 个人备忘录的移动优先首页和桌面双栏延展。
- 快速新建备忘、最近内容、置顶、标签、搜索、回收站入口。
- 自动保存、保存失败提示、加载骨架、空状态和删除恢复提示。
- 暖白极简视觉基线和可打开 HTML 原型。

### 不包含

- 真实后端、账号、同步服务或持久化数据库。
- 跨设备同步、加密、提醒推送和 7 天真实定时清除实现。
- 将当前 Work Item 标记为 completed + sealed。

## 验收标准

- 原型首屏能表达“个人备忘录”的快速记录场景。
- 暖白极简视觉不冷、不像企业后台，也不走糖果风。
- 用户能看到新建、置顶、标签、搜索、回收站和自动保存反馈。
- 空、加载、保存失败状态均有可见表现。
- 390px 移动宽度下无横向溢出，主要操作可触达。
- 桌面宽度下可用左侧列表 + 右侧详情方式浏览。
- `VISUAL_DESIGN.md` 已记录本次已确认视觉基线。

## 当前检查点

实现、静态检查、浏览器验证尝试和视觉基线更新已完成。当前等待用户在 `completion-approval` Gate 确认关闭、继续调整、暂停或拆分 successor。

## 视觉设计上下文

### Visual Seed

用户原始输入：“个人备忘录”。

### 参考图片与解析

- 使用内置参考方向：`playful-consumer-app` 的移动优先、触控友好、亲切空状态。
- 用户随后将视觉收敛为“白色主视觉、干干净净的极简风格”。
- 可借鉴：手机优先、底部导航、轻状态、单列备忘卡片、亲切但不幼稚。
- 不应复制：糖果色大面积铺陈、过度圆角、装饰性图形抢内容。

### 已确认配色

- 候选调色盘对比：A 清亮蓝紫、B 暖黄便签、C 薄荷冷静。用户没有选择候选，而是提出自定义主视觉。
- 用户原始颜色输入：“主色是白色的，干干净净的极简风格”；随后补充担心几乎全白太冷。
- 最终配色来源：custom。
- 用户选择：暖白极简。
- 选择理由：以白色作为主视觉，用微暖背景、暖灰边框和低饱和暖金强调降低冷感，并保持个人备忘录易接受。
- 主视觉色：`#FFFFFF`。
- 页面背景：`#FAF8F3`。
- 主文字：`#24211D`。
- 次文字：`#716B62`。
- 边框：`#E8E1D6`。
- 主操作：`#2B2A28`。
- 次强调：`#F1ECE4`。
- 末级强调：`#D8B56D`。

### 已确认设计禁区

- 不要冷冰冰的纯白医疗感。
- 不要大面积彩色或糖果感。
- 不要蓝紫渐变 AI 工具风。
- 不要复杂卡片堆叠，备忘内容优先。
- 不要过度圆角，保持轻巧但成熟。
- 不要让装饰图形抢走“快速记录”的入口。
- 不要把它做成企业效率后台。
