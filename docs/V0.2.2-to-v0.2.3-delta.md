# v0.2.2 → v0.2.3 差异说明

## 一句话

v0.2.2 解决了 Core 如何被不同 Agent 运行时调用；v0.2.3 解决了真实运行中“用户看什么”和“Adapter 是否严格执行 Core 协议”的问题。

## 新增

- `docs/product/BUSINESS_MODEL_OVERVIEW.md`；
- `BUSINESS_MODEL_OVERVIEW.template.md`；
- `business-model-overview.md` 编译协议；
- 用户只读一个默认文件的交付规则；
- 生命周期对象边界、上下文属性归属、对象身份歧义等校验；
- v0.2.3 评测场景。

## 强化

- Canonical Work Item 文件从“推荐结构”升级为强制合同；
- State Schema 在 Core、Steward、Adapters 间完全一致，运行时元数据只进入 `extensions`；
- Dictionary / Index / Schema 使用 stable ID 并交叉校验；
- Human Decision 必须先压缩再询问，并使用 `BMD-xxx`；
- Core 写入后必须校验、编译总览、再编译其他下游视图。

## 保留不变

- Core / Adapter / Project Assets 三层架构；
- 六种入口模式；
- 14 个子能力；
- Human Decision Control Plane；
- Passive Trigger Interface；
- Agent-neutral 与 runtime-aware；
- UX / Design Engineering 边界；
- 无 CLI、Hook、Plugin、Installer 和外部依赖。
