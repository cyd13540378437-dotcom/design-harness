# Product Business Modeling Examples — v0.2.3

This directory contains two complementary end-to-end examples.

## 1. Work Order Intelligent Dispatch

Path:

```text
work-order-intelligent-dispatch/
```

This is the primary v0.2.3 regression example derived from a real Greenfield modeling run. It demonstrates:

- one default product-owner file: `docs/product/BUSINESS_MODEL_OVERVIEW.md`;
- canonical Work Item files and the frozen State Schema;
- structured Index / Dictionary / Schema contracts;
- full `BMD-xxx` decision compression in the overview;
- stable Action / State / Rule references;
- separate Dispatch Decision and Assignment lifecycles;
- resolution of `Asset / Facility` identity ambiguity;
- contextual placement of arrival cost;
- mandatory consistency validation before overview compilation.

## 2. Seat Pricing Change

Path:

```text
docs/product/
```

This compact passive-trigger example demonstrates an upstream confirmed pricing change, a provisional Seat model, `BMD-001`, canonical evidence and impact reports, and the same single-overview user experience.

In both examples, detailed model and audit files remain available for maintainers and downstream specialists; they are not the default reading burden for the user.
