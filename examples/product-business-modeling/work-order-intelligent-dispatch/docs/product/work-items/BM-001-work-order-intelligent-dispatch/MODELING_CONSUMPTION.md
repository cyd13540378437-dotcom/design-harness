# Modeling Consumption

## Entry Mode

`greenfield_modeling`

## Requested Scope

为设备设施管理行业的工单智能调度产品建立可开发前置业务模型。

## Model Baseline

无既有规范业务模型；存在一份早期实验输出作为 legacy input。

## Consumed Confirmed Sources

- 用户希望开发设备设施工单智能调度产品。

## Consumed Provisional Sources

- 早期实验对象、关系、动作、状态、规则和开发视图；
- 冷启动阶段应先建立最小调度可观测性的分析结论。

## Source Evidence

见 `artifacts/SOURCE_EVIDENCE.md`。

## Assumptions

- 第一版聚焦工程类工单；
- 系统建议 + 人工确认；
- 资源抽象暂时覆盖多种组织资源；
- 策略效果尚未被历史数据证明。

## Missing Inputs

真实排班、历史工单、客户权限规则、策略权重和 SLA 细则。

## Related Triggers

无。

## Explicit Non-scope

数据库字段、API、算法代码、UI 视觉设计和财务收益测算。
