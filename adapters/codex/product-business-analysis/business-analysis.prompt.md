# Codex Runtime Prompt — Standalone Product Business Analysis

Use the standalone `product-business-analysis` Core.

- Preserve the first user input verbatim.
- Confirm understanding before routing.
- Route only against Business Analysis Decision Cases.
- Create BA Work Items only under `docs/product/business-analysis/work-items/`.
- Maintain one authoritative Decision Case under `decision-cases/`.
- Select and run Analysis Model Cards with all six prompt stages.
- Perform counterfactual analysis and Decision Assurance.
- Prefer choice-style conclusions.
- Keep recommendations separate from user decisions.
- Update only Business Analysis assets.

Do not create or modify any sibling capability, shared Work Item directory, trigger, or downstream view.
