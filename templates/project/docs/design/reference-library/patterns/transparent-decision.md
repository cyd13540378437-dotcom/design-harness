# 价格/服务透明决策

## 何时适用

用户需要在价格、费率、时效、限制、配置或风险之间做决定。

## 可借鉴

- 把价格、费用、时效、限制和 CTA 放在同一决策面板
- 关键变化显性化
- 步骤条降低不确定性

## 不应借鉴

- 不要把费用藏到确认后
- 不要用视觉装饰掩盖限制

## QA 检查点

- 费用为 0、异常、优惠、失败时表达清楚
- CTA 前能看到完整影响
- 移动端不折叠关键费用
