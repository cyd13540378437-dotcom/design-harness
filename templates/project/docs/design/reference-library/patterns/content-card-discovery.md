# 内容卡片式发现

## 何时适用

用户需要浏览、比较、收藏或从大量内容/商品/案例中发现目标。

## 可借鉴

- 卡片承载标题、缩略图、关键指标和行动入口
- 列表中只保留最关键的 2–4 个判断字段
- 详情页再展开完整信息

## 不应借鉴

- 不要把移动端瀑布流直接搬到 Web 工作台
- 不要让装饰性图片压过任务行动

## QA 检查点

- 无图、长标题、价格/状态缺失时卡片仍可读
- 桌面端卡片密度与行宽合理
- 移动端点击热区清晰
