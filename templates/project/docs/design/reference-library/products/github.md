# GitHub

## 适合参考的任务

代码协作、评审、issue/PR 流程

## 可借鉴

将状态、评论、文件差异和操作按钮分区；让复杂协作对象可以被逐层扫描。

## 不应借鉴

不要复用第三方品牌资产、图标、插画、文案或完整页面布局；跨终端参考只能抽象借鉴模式，不得直接照搬布局和交互。

## 终端判断

- Primary surface: `web-app`
- Candidate surfaces: `web-app`, `mobile-app`
- 选择它作为参考时，必须先确认当前 Work Item 的目标终端与页面类型是否匹配。

## 备注

补充的 Web/桌面端参考，用于减少资料库对 native mobile app 的偏置。
