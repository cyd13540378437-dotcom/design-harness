# 深海沉浸模式

## 用户可见摘要

适合暗色模式下的专注和高级感。

## 色彩气质

这套色卡的主要气质是：深邃、专注、安静、探索。它在视觉上适合建立清晰的第一印象，同时保留足够的产品可用性边界。用于正式界面时，应把颜色当作产品语义和状态层级的一部分，而不是单纯装饰。

## 适合的产品语义

- 夜间仪表盘
- 阅读
- 研究
- 开发者工具
- 数据探索

## 不适合的产品语义

- 儿童
- 强亲和服务
- 浅色品牌站
- 不适合不做层级控制就直接铺满复杂后台；需要在品牌感和可读性之间做取舍。

## 界面落地建议

适合强入口、发布页、夜间模式和沉浸式产品；慎用于全站长期办公体验。

建议把最深色阶用于文字、导航、主行动或视觉锚点，把中间色阶用于链接、图表和模块重点，把最浅色阶用于页面底色、卡片底色和空状态。彩色面积越大，越需要降低饱和度或提高留白比例。

## Token 转化建议

深海蓝做暗色背景，海洋青做主行动，浅水青只做局部高亮。

可优先映射为：

```yaml
color.primary: primary
color.secondary: secondary
color.accent: accent
color.surface.tint: soft_background
color.background: page_background
```

## 无障碍与可读性风险

风险等级：中。

- 中等风险：浅色底上的彩色小字、细线和禁用态需要单独验证对比度。
- 主 CTA 建议优先使用最深色阶或高饱和强调色，避免只用浅色承担关键行动。

## visual-direction-approval 展示话术

这是一套深邃、专注、安静、探索的配色方向。适合暗色模式下的专注和高级感。适合：夜间仪表盘、阅读、研究、开发者工具、数据探索。不适合：儿童、强亲和服务、浅色品牌站；实现时需要按风险等级检查文字、按钮、边界和状态反馈的对比度。
